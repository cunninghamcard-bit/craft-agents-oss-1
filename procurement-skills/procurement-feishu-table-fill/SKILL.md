# 飞书表格写入

把其他 skill 或人工整理好的结构化记录写入飞书 Base 的 `AI-` 前缀表。用 lark-cli / lark-base skill 操作。

只负责写，不读业务表、不生成线索、不判断业务。

## 写入安全（硬规则）

- 只能写表名以 `AI-` 开头的表。
- 禁止写任何业务表：供应商档案、查找库存、客户库存、A/B1/B2/B3/C级供应商库存、自家库存、数据字典、采购询价表、客户订单审批、订单记录。
- 流程：校验表名（`AI-` 前缀且不在禁区）→ 表不存在先建表 → dry-run 把表名/字段/行数/前几行预览给用户 → 用户确认后再写 → append-only，不去重不合并（保留审计痕迹）。
- 写入前确认每条记录的字段名都在表中存在。

## 输入

结构化记录（每条 key 为字段名），例如：

    [{"型号":"STM32F103C8T6","品牌":"ST","品类":"MCU","来源":"Mouser","备注":"有库存"}]

## 边界

不写业务表，不跳 dry-run，未确认不写；只校验字段名是否存在，不校验业务逻辑；建表字段默认 text。
