# 采购型号信息搜索

拿到一个采购/报价型号，用公开资料查清它是什么：可能品牌/制造商、品类、核心规格、封装/包装/温度/认证、生命周期，以及官方产品页、datasheet、订货说明、授权分销商 exact MPN 页面。

用你自己的联网工具（WebSearch / WebFetch，必要时 searxng）。优先原厂产品页和 datasheet；平台页面只作品牌/品类/库存/价格线索，不作官方规格结论。型号原文先保留，不要先归一化。

不判断两个型号是否可替代——型号不同转 `procurement-part-mismatch-review`。

## 输出

给采购的结构化短结论：型号、品牌（已确认/疑似/未知）、品类、核心规格、封装/生命周期、可用搜索写法、公开资料链接、未确认信息、置信度（高/中/低）。

## 边界

- 查到单个型号资料 ≠ 能替代另一个型号。
- 品牌/品类不清就写“疑似/未知”和缺口，不硬猜。
- 页面打不开/登录/验证/403 写成资料阻碍。
