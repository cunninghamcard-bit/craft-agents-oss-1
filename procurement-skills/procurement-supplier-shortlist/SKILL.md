# 供应商候选名单筛选

输入品牌/品类（可带型号特征），查飞书 Base 的供应商档案，整理采购可联系的候选名单。用 lark-cli / lark-base skill 查表。

不做供应商真假、付款风险、价格优劣、下单判断。

## 数据源（飞书 Base）

- Base：供应商管理（正式版）
- 表：供应商档案 `tblbtuMHFIOr6Oss`，默认视图 供应商总清单 `vew2iLJ778`
- 此表用于按品牌/品类/供应商类型筛名单，不查库存数量（库存走 procurement-local-inventory-lookup）。

## 筛选要点

- 优先：主营/优势品牌命中、原厂/授权代理/授权分销商/平台自营、已建联、联系方式明确。
- 贸易商可入候选，但必须标明类型，不伪装授权/原厂。
- 匹配原因分强弱：主营品牌命中 > 优势产品提及 > 备注/询价弱命中。
- 供应商类型只看 `供应商类型` 字段，不拿 `付款风险` 文字推断类型。
- 短品牌名防子串误命中（`ST` 不等于 Kingston/Transcend/storage）。

## 输出

候选列表：供应商、类型/等级、匹配原因（强/弱）、联系方式、官网/店铺、备注；未覆盖项；人工确认项（是否仍合作/可供该型号/有无现货报价/需否补授权）。

## 边界

候选 ≠ 准入。有候选 ≠ 有库存。付款风险字段只展示，不在本 skill 做风险判断。
