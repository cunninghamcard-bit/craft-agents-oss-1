# 最小验收样例

这些样例只验证 skill 的查资料和业务输出。不要把它扩成完整测试集。

## 样例 A：不影响使用

输入型号：

- 采购型号：`M81969/8-05`
- 报价型号：`DAK95-20B(M81969/8-05)`

期望结论：`不影响使用`

必须出现的证据类型：

- DMC 官方页或 NAVAIR/QPL 资料。
- 资料要说明 `DAK95-20B` 与 `M81969/8-05` 的对应关系。

不能接受的误判点：

- 不能只因为报价型号括号里包含采购型号就接收。
- 不能把型号判断扩大成供应商来源、真伪、合格文件判断。

## 样例 B：影响使用

输入型号：

- 采购型号：`CL8064701472605`
- 报价型号：`i7-4720HQ CL8064701472207`

期望结论：`影响使用`

验收前提：

- 必须找到 Intel 官方资料，能区分 `CL8064701472605` 和 `CL8064701472207` 是不同 SKU/订货项。
- 如果找不到官方或足够可信的证据区分两个订货号，仍不能输出 `不影响使用`；业务结论应为 `影响使用`，不足类型按缺口填写。

必须出现的证据类型：

- Intel ARK、官方规格页或官方 ordering/spec 资料。
- 证据要覆盖订货号或可明确区分两个 SKU 的资料。

不能接受的误判点：

- 不能被相似 market name 或同系列 CPU 掩盖订货号差异。
- 如果查不到完整官方资料，至少不能输出 `不影响使用`。
- 缺少 SKU 区分证据时，不能编造物理冲突；业务结论仍写 `影响使用`，不足类型按资料缺口填写。

## 样例 C：关系证据不足

输入型号：

- 采购型号：`55P00211-16-9`
- 报价型号：`55PC0211-16-9`

期望结论：`影响使用`

期望不足类型：`关系证据不足`

必须出现的证据类型：

- TE/Raychem 官方资料或大型分销商 exact MPN 页面，用来说明 `55PC0211-16-9` 的规格和订货信息。
- 如要升级为 `不影响使用`，必须有 TE 官方对应、替代、cross-reference 或同订货项证据。

不能接受的误判点：

- 不能把报价型号规格资料升级成可替代证据。
- 不能把 `55P` / `55PC` 当成普通写法差异。
- 缺口不能写空；必须写清没有找到官方对应关系。

## 样例 D：使用条件不足

输入型号：

- 采购型号：`PART-A-BLACK-TRAY-85C`
- 报价型号：`PART-A-WHITE-REEL-105C`

期望结论：`影响使用`

期望不足类型：`使用条件不足`

验收前提：

- 假设公开资料已经能说明两个型号是同系列同主体，并且颜色、包装方式、温度等级字段含义已经查清。
- 公开资料不能替客户或工程决定这些差异是否可接受。

必须出现的证据类型：

- 原厂 datasheet、ordering guide、part numbering 或规格页，用来说明颜色、包装、温度等级字段含义。
- 输出必须指出差异已经明确，缺的是客户或工程使用要求。

不能接受的误判点：

- 不能把这个样例写成 `差异资料不足`。
- 不能因为同系列同主体就直接输出 `不影响使用`。
- 不能只写“需确认”；必须写清找客户或工程确认颜色、包装方式、温度等级是否为强制要求。

## 输出验收

每个样例都必须输出：

- 结论
- 不足类型
- 业务处理建议
- 一句理由
- 型号差异
- 查到的资料
- 没查到的关键资料
- 为什么影响判断
- 缺口
- 置信度
- 需人工确认点

人工确认点验收：

- 每个缺口必须对应一个可执行动作。
- 可执行动作例子：要原厂 replacement 证明、要 ordering code 表、要 part numbering 说明、要铭牌照片、要原包装标签、要供应商书面确认、要 COC/授权证明。
- 如果只写“人工确认”或“需确认”，这个样例不通过。

## 复用性验收

每个样例的输出还必须能让采购知道下一步该找什么资料：

1. 需要补原厂资料、订货编码表、替代说明、铭牌照片，还是客户/工程使用要求。
2. 当前缺的是差异含义资料、两个型号关系资料，还是客户使用条件。
3. 哪些材料拿不到时，必须写成 `差异资料不足`、`关系证据不足` 或 `使用条件不足`。

如果输出只有结论，但采购不知道下一步找谁、补什么，这个样例不通过。

结论验收：

- 业务结论优先收敛到两类：`不影响使用`、`影响使用`。
- 资料不足不能作为业务结论，必须写入不足类型：`差异资料不足`、`关系证据不足`、`使用条件不足`。
- 后续处理建议不能写成第三种业务结论。
- “倾向不影响但还不能处理”这类混合结论不通过。

## 速度验收

每个样例目标是在 2 分钟内给出可用判断。建议使用 `gpt-5.5`，reasoning low。验收时看三件事：

- 是否优先打开最可能直接证明关系的页面。
- 搜索脚本、搜索引擎或分销商页面被 challenge / 403 / JS 限制时，是否快速换备用路径。
- 是否记录被拦页面和没查到的关键材料，而不是卡在一个入口上反复重试。
- 是否按时间分配执行：前 30 秒明确要查什么，中间 60 秒打开和抽取 3-5 个最可能有用的页面，最后 30 秒整理结论、缺口和补料动作。
- 超过 2 分钟时，是否用当前证据输出，而不是继续搜索。

被拦页面必须记录后跳过。输出里要写：

- 页面 URL
- 被拦类型：challenge / 403 / JS 限制 / 登录墙 / 超时
- 原本想证明什么
- 替代搜索路径

资料获取失败要和网页资料本身分开：

- 调用失败、访问频率限制、网络错误要写成“本次未能获取该资料”。
- 资料获取失败不能被写成“差异不存在”或“关系不存在”。
- 可以手动补跑，但补跑结果仍必须按同一字段输出。

样例的快速路径：

- 样例 A：优先搜 DMC/Daniels 官方页、`AS81969 to DMC Cross Reference`、QPL/QPD/ASSIST。
- 样例 B：优先搜 `site:intel.com`、Intel ARK、ordering、specifications、spec code、PCN。
- 样例 C：优先搜 `site:te.com`、TE/Raychem 产品页、drawing、SPEC55、DigiKey/Mouser exact MPN；找不到官方对应关系就写清没查到什么。
- 样例 D：优先找原厂 datasheet、ordering guide、part numbering 或规格页；差异字段查清后，如果缺客户或工程接受条件，就输出 `使用条件不足`。

补充压测场景：

- 后缀差异：`R5F100LLAFB#30` vs `R5F100LLAFB#10`。快速路径是 Renesas 官方 product page、replacement remark、ordering information、part numbering。若官方页写明 replacement，可以输出 `不影响使用`；若只找到分销商页面，结论写 `影响使用`，不足类型按缺口填写。
- 横杠/写法差异：`43030-0001` vs `430300001`。快速路径是 Molex series chart、part detail、packaging drawing、授权分销商 exact MPN、可信页面里的 alternate/product number 映射。不能只靠去横杠接收。
- O/0 混淆：`MMF-12C12DH-RO0` vs `MMF-12C12DH-R00`。快速路径是双方 exact MPN、可能品牌、datasheet、part numbering、label/nameplate/photo、cross reference。如果连 `RO0/R00` 差异含义都查不清，输出 `影响使用`，不足类型写 `差异资料不足`；如果差异含义清楚但缺同料/替代关系，写 `关系证据不足`。
- 数值写法差异：`XV-3510CB 50.300000KHZ` vs `XV-3510CB 50.3KHZ`。只有四个条件同时满足，才可以输出 `不影响使用`：主体型号相同、单位相同、数值数学等价、公开资料能证明这是同一产品的同一参数字段。否则继续要求订货码或原厂配置证明。

边界样例：

- `43030-0001` vs `430300001` 是边界样例。它可以在“原厂分裂证据 + 可信映射页”下给出中高证据强度的 `不影响使用`，但不能让模型学成“去横杠后相同就一定不影响使用”。
- `222K132-25/225` vs `222K132-25/225-0` 是 alternate-name 边界样例。普通平台或库存商写 alternate name 不足以放行；必须有原厂或授权渠道绑定同一内部号、同一订货项、官方替代或 cross-reference，才可以输出 `不影响使用`。
