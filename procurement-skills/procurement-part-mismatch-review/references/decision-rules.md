# 判断规则（规格对比驱动）

## 三类结论

- **能用**：报价型号在所有关键规格上覆盖采购型号要求；或有原厂/授权渠道的明确替代、cross-reference、同订货项声明。
- **不能用**：任一关键规格冲突，报价型号不满足采购型号要求。
- **需补料**：关键规格查不到，无法确认。细化缺口：`缺规格资料` / `缺替代关系证据` / `缺客户使用要求`。

不要把"资料不足"写成第三种业务态度——它就是"需补料"，并说明缺哪类。

## 关键规格按品类定

判断"关键"看采购用途。常见关键项：

- **封装 / 引脚 / pin 定义**（pin 不兼容直接不能用）。
- **电参数**：工作电压范围、电流、精度/容差、容值/阻值/功率、频率/速度。
- **Memory / 接口 / 协议**。
- **温度等级**（商业/工业/车规）、**认证**（AEC-Q、UL、3C 等）。
- **生命周期**（替代本身停产就没意义）。

## 覆盖 vs 冲突

- **覆盖（通过）**：报价型号规格 ≥ 采购要求——温度范围更宽、精度更高、电压范围覆盖、容值/阻值相同、pin 相同。
- **冲突（不能用）**：报价型号不满足——封装不同、电压不够、精度更差、温度更窄、pin 不同、容值/阻值不同。
- **数值参数**注意单位与等价：`10uF = 10µF`（同）；但 `±10% vs ±20%` 是精度差异，要看采购是否接受（接受→通过，未知→需补料/缺客户使用要求）。

## 证据强弱

- **强**：原厂/授权渠道的明确替代声明、cross-reference、同订货项、官方对比表（Octopart Alternate Parts 的 SiliconExpert 规格对比也算结构化强证据）。
- **中**：双方原厂 datasheet 分别给出规格，靠规格逐项对比得出结论。
- **弱**：只有分销商/平台/聚合站的"相似/替代"提示，没规格支撑——不足以下"能用"，按需补料处理。

## 注意

- 别只凭型号字符串相近、封装相近放行，必须有规格支撑。
- 写法/别名差异（去横杠、大小写、空格）只是搜索线索，要有资料证明两种写法指向同一实物。
- 报价页/分销商页标了 alternate/similar 不等于替代成立，要回到规格或原厂替代声明。
- 替代最终能不能用常要客户/工程确认 pin 兼容、认证、是否接受差异——这写进"需补料 / 人工确认项"，不是第三种结论。
