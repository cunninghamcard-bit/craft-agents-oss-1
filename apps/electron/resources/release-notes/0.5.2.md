## What’s New in 0.5.2

### Explore mode: safer review workflow defaults
- Added a built-in **safe review baseline** for Explore mode so common validation stays non-destructive but practical.
- Default allowlist now explicitly includes common typecheck commands:
  - `bun run typecheck`
  - `bun run typecheck:all`
  - `bunx tsc --noEmit`
  - `tsc --noEmit`
  - package manager aliases (`npm run typecheck`, `yarn typecheck`, `pnpm typecheck`)
- Existing read-only search commands (`grep`, `rg`, etc.) remain supported.

### Session tool reliability hardening
- Improved consistency for feature-flagged session tools across backends (Claude, Pi, session-mcp-server).
- Tool listing and tool execution now use aligned filtered definitions to prevent backend drift.

### Debug/dev feature flag robustness
- Feature flag runtime detection is now evaluated in a runtime-safe way to avoid import-order issues in debug sessions.
- Added shared runtime helper behavior used across components for consistent dev/debug flag decisions.

### Typecheck coverage
- Updated `typecheck:all` to include `session-tools-core`, reducing the chance of hidden regressions in shared tooling logic.
