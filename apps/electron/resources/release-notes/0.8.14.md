# v0.8.14 — Telegram supergroup topics for automations

## Features
- **Telegram supergroup pairing** — Pair a Telegram supergroup to a workspace from **Settings → Messaging → Telegram → Pair Supergroup**. The bot captures the supergroup's `chatId` after you type the pairing code in any topic, after which the workspace's adapter accepts messages from that supergroup alongside DMs. Bound bindings now display as `Group › Topic #N` in the Settings list and `list_messaging_channels` tool output.
- **Per-automation Telegram topic routing** — Add the optional `telegramTopic` field to any automation matcher in `automations.json` to route the sessions it spawns into a dedicated forum topic in the paired supergroup. The topic is **created on first use** and **reused** thereafter; multiple matchers using the same value share one topic. The Automation Info page shows the configured topic with a status hint, and the Settings → Messaging panel surfaces a one-line note when a supergroup is paired.

```json
{
  "matcher": "^urgent$",
  "telegramTopic": "Urgent Alerts",
  "actions": [
    { "type": "prompt", "prompt": "Investigate the urgent issue: $LABEL" }
  ]
}
```

## Improvements
- **Threaded inbound + outbound** — The Telegram adapter now captures `message_thread_id` on incoming text, attachments, and callback queries, and forwards `threadId` on every outgoing send. Bindings persist `threadId`, so different topics in the same supergroup are independently bindable and one channel can host multiple sessions across topics.
- **Documentation parity** — The new field is documented in the public docs site (`apps/online-docs/automations/overview.mdx`), the in-app docs viewer (`apps/electron/resources/docs/automations.md`), and the global agent docs (`~/.craft-agent/docs/automations.md`). Translations for `automations.labelTelegramTopic`, the two routing hints, and the Settings note are included for all 7 supported locales.

## Bug Fixes
- None.

## Breaking Changes
- None. The `telegramTopic` field is fully optional. Workspaces with no paired supergroup behave exactly as before.

## Known Limitations
- If the bot lacks the **Manage Topics** permission, topic creation fails on every run and the session runs unbound. A future release will probe the permission during pairing.
- If a topic is deleted in Telegram, the cached entry becomes stale and sends fail with `MESSAGE_THREAD_NOT_FOUND`. Recovery is manual today (rename the matcher's `telegramTopic` to force a new topic).
- The Telegram General topic and DMs are not affected by the new routing — both continue to follow the existing pairing flow.
