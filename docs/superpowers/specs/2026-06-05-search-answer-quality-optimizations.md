## A. 搜索过程(search behavior)

### A1 ✅ 渐进式搜索:先钉死平台,不够再扩
- **改什么**: 搜索分级。第一档只搜**固定的领域平台白名单**;命中不足/质量差,才放开到通用 web。
- **为什么**: 当前是单查询、满世界搜、默认 5 条(`create-search-tool.ts:70`),垂直问题命中率低。agentic search 的核心就是"用结果质量驱动下一步"。
- **怎么落**: SearXNG 支持按 `engines`/域名限定;在搜索工具里加"档位"参数 + system prompt 指令"先搜 X 平台,不够再 broaden"。**具体钉哪些平台 = 待你给。**

### A2 ✅ 搜索结果必须引用网址链接
- **改什么**: 凡用到搜索/网页内容的结论,**必须在答案里带出处 URL**(行内链接或末尾来源列表)。
- **为什么**: 现在搜索把 URL 给了模型(`create-search-tool.ts:24-48`),但 system prompt **没有一句**要求引用(`system.ts`)→ 拿了 URL 不引用 → 易"丢源"瞎编。grounding+引用可把幻觉降 42–68%。
- **怎么落**: system prompt 加强制引用指令 + 搜索工具描述里点明"答案需引用返回的 URL";可在 eval 里用代码判"是否带链接"。

### A3 ☐ 读全文,不只看摘要
- **改什么**: 对 top N 命中,自动 `web_fetch` 取全文再下结论,而不是只拿 50–300 字摘要硬答。
- **为什么**: 现在 `web_search` 只回摘要(`searxng.ts:47` 截断 300 字),全文要模型自己再调 `web_fetch`,而 prompt 几乎没推它。碎片上推理 = 答不全/答错。
- **怎么落**: prompt 指令"关键问题先 fetch top 1–3 全文";或在搜索后自动并行 fetch。

### A4 ☐ 答前自评证据是否充分,不够就再搜/改写
- **改什么**: 回答前先判"现有证据够不够"。不够 → 改写 query / 换平台 / 再搜一轮,而不是硬凑一个答案。
- **为什么**: 信息检索 agent 的核心能力之一就是"自评证据充分性"(epistemic competence)。现在没有这一步。
- **怎么落**: system prompt 加自检指令;eval 里看"证据不足时是否继续搜 vs 直接编"。

### A5 ☐ 搜索失败/兜底要透明,别偷偷用训练记忆答
- **改什么**: 搜索失败或 fallback 时,**明说**"没搜到/降级了",不要假装用内部知识答得很笃定。
- **为什么**: 代码现状是 provider 失败静默 fallback 到 DDG;模型可能直接用训练记忆答(尤其价格/库存这种会过时的),用户无感知。
- **怎么落**: 工具层把 fallback 状态透传给模型 + prompt 指令"搜索没结果时如实说明"。

## B. 回答行为(answer behavior)

### B1 ☐ 区分"有来源支撑" vs "模型推测"
- **改什么**: 有出处的当事实给(带链接);没把握/纯推测的**明确标注**"这是推测,未核实"。
- **为什么**: calibration——降幻觉的关键不是更自信,是"知道自己不知道"。2026 实践:"reflect on your answer, consider you might be wrong"。
- **怎么落**: system prompt 指令 + eval 用 LLM-judge 判"是否把无源断言当事实"。

### B2 ☐ 时效性:会过时的信息优先近期来源 + 带日期
- **改什么**: 价格、库存、规格、行情这类**会变**的信息,优先近期来源,并在答案里标来源日期。
- **为什么**: 采购/元器件类查询对时效极敏感,旧数据 = 错数据。
- **怎么落**: 搜索带时间倾向 + prompt 指令"时效敏感信息须标日期/时间窗"。

## C. 关联项(属系统层,可单独立项)

### C1 ☐ 技能自动触发(不靠用户手敲 `[skill:slug]`)
- **改什么**: 启动时把所有 skill 的 name+description 预加载进 prompt,让模型**自动选用**。
- **为什么**: 现在只有用户点名 `[skill:slug]` 才触发(`system.ts:500-507`)→ 装的一堆飞书/采购 skill 等于没有。这是全系统最大的浪费。
- **怎么落**: 系统层改加载逻辑 + 按 Anthropic 最佳实践调 description(第三人称、写明触发场景、稍"pushy")。
- **注**: 这条收益最大但不属"搜索",是否纳入本清单由你定;否则单独立项。
